Am Mittwoch 10 Januar 2007 19:42 schrieb Wolfgang Becker:
> I'm not so sure if that is a good idea. Does that mean that other
> hardcoded values like max_drop_height for some badguys or the
> X_OFFSCREEN_DISTANCE have to be adjusted? No time to test it
> now, so I did not commit this patch yet.

Theoretically it should work. There is a guaranteed 600x600 square in logical 
coordinates.
The level distances are in logical coordinates, so they are not affected.
The OFFSCREEN_DISTANCE has some bugs at very extreme aspects (like 16:5  -  
SCREEN_WIDTH > X_OFFSCREEN_DISTANCE)

I have another patch to correct some calculations and fix that problem:
http://www.unix-ag.uni-kl.de/~kldenker/supertux/offscreen.patch

The OFFSCREEN_DISTANCE-constants are not changed.

This patch is also useful without the widescreen-patch.
