I agree with R A on that one. In M1, I mapped my keys so that up meant jump
inlevel, and move up on worldmap too. Then space meant special/run. In SVN,
I tried to do that too, but it wouldn't let me. My vote is to allow the
option to make the up/jump keys the same even if you don't find it a sane
keymapping. I also think up shouldn't affect doors, but action should.
Action makes more sense to me, as tux is in a 2D world, and therefore up
should mean Y axis, not Z axis.   Of course, having those keys the same
might cause programming problems... I'm not sure.
