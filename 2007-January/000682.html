On 30-Dec-06, at 4:52 PM, Jayson Kempinger wrote:

> There are a few bugs ... and a limitation that forces the app  
> bundle to be named SuperTux.app

The first of the arguments passed to main() should be the full path  
to the binary, ie "/Applications/Games/SuperTux.app/Contents/MacOS/ 
SuperTux", as long as it was launched from the Finder.  You can trim  
of the end of the string to find the path the the Resources folder  
and the data folder inside.  This should allow you to work around the  
limitation.
